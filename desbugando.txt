A thread input_handle vai deixar o mutex lockado enquanto ela espera que a palavra termine.
Enquanto o mutex estiver travado a palavra atual tem que cair na tela, isso vai ser feito atraves de uma outra thread dentro da graphical handle. Essa thread vai printando a palavra caindo na tela enquanto o mutex esta travado. Na hora que o mutex ser liberado essa thread vai ser morta, comecando logo em seguida de novo com uma nova palavra

------------------------------------------------------------
Perfumarias:

1) A palavra vai mudando de cor conforme cai:
1/3 = verde
2/3 = amarelo
3/3 = vermelho

Criar uma array com os numeros das cores em ANSII
Percorrer essa array com um for com um "step" (passo)
que seja multiplo do valor maximo de linhas, para que ela
percorra por todas as cores.

2) Adicionar dificuldade
Basicamente a dificuldade é o tempo em milisegundos da atualizacao da letra na tela. Se eu deixar isso global e alterar em um menuzinho antess do programa começar a dificuldade fica dinâmica.

3) Placar de acertos
Placar que conta quantas palavras você acertou. 

4) Menu principal estilizado
Usar arte ASCII para criar um menu estilizado onde o jogador vai poder escolher a dificuldade, sair ou iniciar o jogo.
